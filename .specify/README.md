# Spec-Kit Support

This repository uses Spec-Kit for spec-driven development.

## Purpose
Provide a clear path for writing and tracking feature specs before implementation.

## Directory Layout
- `.specify/` contains spec guidance and reusable templates
- `specs/` contains tracked feature specifications

## Getting Started
1. Copy `.specify/templates/feature-spec-template.md`
2. Rename the copy to the next `BCCA-FEAT-XXX-*.md`
3. Fill in goals, acceptance criteria, and success metrics
4. Keep the spec committed in `specs/`
